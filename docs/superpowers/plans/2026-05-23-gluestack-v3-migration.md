# GlueStack UI v1 → v3 Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the Expo React Native app from GlueStack UI v1 token-prop styling to GlueStack UI v3's NativeWind (Tailwind CSS) approach.

**Architecture:** Install NativeWind + Tailwind CSS, use the GlueStack CLI to generate local v3 components, then rewrite App.tsx to use className-based styling instead of token props. Single-file app so blast radius is small.

**Tech Stack:** Expo SDK 54, React 19, React Native 0.81, NativeWind v4, Tailwind CSS v3, GlueStack UI v3

---

## File Structure

| File | Action | Purpose |
|------|--------|---------|
| `package.json` | Modify | Add NativeWind/Tailwind deps, remove old GlueStack deps |
| `tailwind.config.js` | Create | Tailwind configuration with NativeWind preset |

| `nativewind-env.d.ts` | Create | TypeScript support for NativeWind className prop |
| `components/ui/*` | Create (CLI) | GlueStack v3 local components generated by CLI |
| `App.tsx` | Modify | Update imports, provider, and convert all styling |

### Task 3: Configure NativeWind in Babel and TypeScript

**Files:**
- Create: `babel.config.js`
- Create: `nativewind-env.d.ts`
- Modify: `tsconfig.json`

- [ ] **Step 1: Create `babel.config.js`**

This project has no `babel.config.js` in the root. Expo uses `babel-preset-expo` by default, so we add it explicitly along with `nativewind/babel`.

```js
module.exports = function (api) {
  api.cache(true);
  return {
    presets: ['babel-preset-expo', 'nativewind/babel'],
  };
};
```

- [ ] **Step 2: Create `nativewind-env.d.ts`**

This adds TypeScript support for the `className` prop on React Native components.

```ts

```

### Task 8: Convert pillVariant and Badge styling

**Files:**
- Modify: `App.tsx:103, 482-490`

- [ ] **Step 1: Update the `pillVariant` function**

The `pillVariant` function returns GlueStack token strings used as `bg` prop on `Badge`. In v3 NativeWind, we use Tailwind color classes instead.

Before:
```tsx
const pillVariant = (ok: boolean) => (ok ? '$success600' : '$backgroundDark500');
```

After:
```tsx
const pillVariant = (ok: boolean) => (ok ? 'bg-green-600' : 'bg-gray-600');
```

- [ ] **Step 2: Update Badge components to use className**

Before (line 482):
```tsx
<Badge bg={pillVariant(Boolean(scannedApi))}>
  <BadgeText color="$textLight0">
```

After:
```tsx
<Badge className={pillVariant(Boolean(scannedApi))}>
  <BadgeText className="text-white">
```

Apply the same change to the second Badge (line 487):
```tsx
<Badge className={pillVariant(Boolean(scannedInfo))}>
  <BadgeText className="text-white">
```
